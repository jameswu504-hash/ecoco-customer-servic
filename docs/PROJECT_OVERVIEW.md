# ECOCO AI 客服專案總覽

本文件提供 ECOCO AI 客服系統的高層次總覽，協助內部主管、客服、營運與維護者快速理解系統定位、功能範圍、資料流程與維護方向。

## 1. 專案定位

ECOCO AI 客服是一套面向使用者的 AI 客服系統，並搭配內部後台進行知識庫維護、對話查看、評分追蹤、知識缺口管理與營運報表整理。

此系統的核心不是讓 AI 自由回答，而是讓 AI 依據 ECOCO 已整理、可追蹤、可更新的知識庫回答。系統會透過 RAG 檢索 PostgreSQL 中的知識資料，再由 Claude 依照品牌語氣與風險規則產生客服回覆。

## 2. 系統目標

| 目標 | 說明 |
| --- | --- |
| 即時回答常見問題 | 使用者可快速詢問點數、站點、回收規則、App 與優惠券問題 |
| 統一客服口徑 | AI 回答依據整理後的 ECOCO 知識庫，不依賴模型自由發揮 |
| 降低高風險回覆 | 對補點、退款、帳號、客訴等問題套用保守規則 |
| 建立知識維護流程 | 內部人員可新增、修改、封存與匯出知識 |
| 支援營運觀察 | 後台提供客服量、問題分類、評分與知識缺口 |
| 支援交接 | 透過文件、Git JSON 與 PostgreSQL 流程保留可交接資料 |

## 3. 主要功能

| 功能 | 說明 |
| --- | --- |
| 客服前台 | 使用者提問與 AI 回覆頁面 |
| 常見問題快捷入口 | 提供回收點數、點數期限、合作商家、投瓶規則、找站點、聯絡客服等入口 |
| AI 回覆 | Claude 根據 RAG 找到的 ECOCO 知識產生回答 |
| 高風險 guardrail | 依 `risk_level` 與回覆政策限制 AI 不做未確認承諾 |
| 對話紀錄 | 保存使用者與 AI 的互動，供後台檢視 |
| 使用者評分 | 追蹤「有幫助」與「需改善」回饋 |
| 知識缺口 | 紀錄 AI 無法確定回答的問題，作為補知識待辦 |
| 知識維護 | 後台新增、修改、封存、恢復與匯出知識 |
| 主管報表 | 提供客服量、問題分類、待處理項目與週報/月報素材 |

## 4. 系統流程

```text
使用者提問
  -> 前台送出 /api/chat
  -> 後端驗證輸入並建立 session
  -> RAG 從 PostgreSQL knowledge_chunks 找相關知識
  -> 套用 risk_level 與 response policies
  -> Claude 產生回答
  -> 回覆使用者
  -> 寫入 conversations / ratings / unanswered_questions
  -> 後台供客服與維護者追蹤
```

## 5. 資料來源

目前整合的資料來源包含：

- ECOCO 官網 FAQ
- Excel FAQ 與客服回覆素材
- Meta AI 指令與品牌語氣規範
- 舊 CommandCenter 專案可參考內容
- 內部 SOP 與回覆政策
- 營運、客服與主管確認後的補充資料

舊資料只能作為比對來源，不應未經整理直接匯入。正式 AI 回答應以 PostgreSQL 知識庫與 Git JSON 版本資料為準。

## 6. 內部角色分工

| 角色 | 主要責任 |
| --- | --- |
| 客服人員 | 查看 AI 回覆、評分與知識缺口，補充已確認的常見問題 |
| 營運窗口 | 確認站點、機台、活動、合作方與現場狀態資訊 |
| 客服主管 | 決定知識缺口優先順序，確認高風險回覆與客服口徑 |
| 系統維護者 | 維護資料庫、Git JSON、部署、文件與例行檢查 |
| 工程協作者 | 維護程式、RAG、API、資料庫 schema 與安全機制 |

## 7. 知識管理方式

系統採用兩層知識管理。

| 層級 | 位置 | 用途 |
| --- | --- | --- |
| 線上執行資料 | PostgreSQL `knowledge_sections` | AI 目前實際使用的知識 |
| 正式版本資料 | `data/ecoco-knowledge-import.json` | 可備份、交接、重新匯入的知識包 |

後台修改會立即寫入 PostgreSQL，但不會自動寫回 GitHub。重大改版、交接或正式備份前，應從後台下載 JSON，人工確認後更新 Git。

## 8. 目前維護重點

- 確認 OpenAI embedding 額度與 `OPENAI_API_KEY` 設定，使語意檢索穩定運作。
- 持續清理重複、過期或衝突知識。
- 設定對話紀錄保存與個資治理原則。
- 讓客服人員能以後台完成日常知識維護。
- 讓主管能以報表掌握客服問題趨勢。
- 讓文件與實際系統保持一致，降低交接成本。

## 9. 專案原則

ECOCO AI 客服的長期價值不只在於自動回答，而在於建立一套可被內部維護的客服知識系統。

所有後續調整應符合：

- 資料來源清楚。
- 回覆風險可控。
- 知識更新可追蹤。
- 版本資料可備份。
- 文件流程可交接。
